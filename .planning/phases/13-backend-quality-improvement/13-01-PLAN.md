### buildStockSummary Optimization (D-02, D-03)

**Current inefficient pattern (lines 134-152):**
```typescript
const stockTagsByLocation = await this.prisma.tag.findMany({
  where: { deletedAt: null, status: { in: [TagStatus.IN_WAREHOUSE, TagStatus.IN_WORKSHOP] }, locationId: { not: null } },
  select: { locationRel: { select: { type: true } } },
});
const locationTypeCounts = stockTagsByLocation.reduce<Record<string, number>>((acc, row) => {
  const locationType = row.locationRel?.type;
  if (!locationType) return acc;
  acc[locationType] = (acc[locationType] || 0) + 1;
  return acc;
}, {});
```

**Target efficient pattern (groupBy at DB level):**
```typescript
const locationGrouped = await this.prisma.tag.groupBy({
  by: ['locationId'],
  where: { deletedAt: null, status: { in: [TagStatus.IN_WAREHOUSE, TagStatus.IN_WORKSHOP] }, locationId: { not: null } },
  _count: { _all: true },
});
// Then fetch locations to get type for each locationId
const locationIds = locationGrouped.map(g => g.locationId).filter(Boolean);
const locations = await this.prisma.location.findMany({ where: { id: { in: locationIds } } });
const locationMap = new Map(locations.map(l => [l.id, l.type]));
const locationTypeCounts = locationGrouped.reduce<Record<string, number>>((acc, g) => {
  const type = g.locationId ? locationMap.get(g.locationId) : undefined;
  if (type) acc[type] = (acc[type] || 0) + g._count._all;
  return acc;
}, {});
```

Note: Prisma groupBy does NOT support include for nested relations. Must fetch locations separately.

const locationGrouped = await this.prisma.tag.groupBy({
